<H1><span style="color: #FFF;">SCP-ğ•</span></H1>
<H3><span style="color: #FFF;">(Formerly SCP-ï¿½ï¿½ï¿½ï¿½)</span></H3>
<p><b><span style="color: #FFF;">Object class: Euclid</span></b></p>

<div class="block grey" data-title="SECURE CONTAINMENT PROCEDURES"><p>No numerals, ordinals, or other words indicative of a singular, finite value may be used in any documentation related to this anomaly. SCP-ğ• is not to be assigned any numerical metrics, even for cataloguing purposes, including Disruption and Risk Classes. This also includes the standard numerical classification, which has been replaced with the Unicode character described as "Blackboard Bold Uppercase Y" (ğ•).</p>

<p>Information about this anomaly must be stored inside a virtual software box built to utilize a non-numerical coding system and rendering engine specifically built for this purpose, known as the RGU-ğ• system. This system is to be installed at Foundation Infohazard Containment Site-â™ƒ, which specializes in containment via alternative-text and non-textual methods.</p>

<p>The knowledge of the nonexistence SCP-ğ•-A is to be monitored in the general population. Individuals who discover the nonexistence of SCP-ğ•-A must be treated with amnestics. The use of memetic textual deterrents ("<a href="http://www.scp-wiki.net/numbers-lie">mipsum</a>") may be utilized to hide discovered discrepancies.</p>

<p>Within the period approximately equal to a full rotation of the Earth, The RGU-ğ• system must be activated, which will generate a random number of sufficiently large value as to be statistically improbable that this number will be utilized for any purpose by humans in the foreseeable future. This number will then be added to the end of this containment file.</p>

<p>If several days pass without being "fed", SCP-ğ• consume a number at random, resulting in potential catastrophic consequences for financial, computing, and other mathematics-derivative fields of science.</p>

<p>In effect, the processes protecting this containment file are the same processes containing SCP-ğ•.</p></div>

<div class="block grey" data-title="DESCRIPTION">SCP-ğ• is a finite mathematical set of numbers. The most accurate definition of SCP-ğ• can roughly be defined as "All numbers which do not exist."</p>

<p>SCP-ğ•'s existence is intrinsically tied to baseline reality. The anomalous effect of RGU-ğ• manifests when numbers of any value are associated with it. When a number is exposed to SCP-ğ•, the number is added to the set in question, causing it to cease existing. A number which ceases to exist is unable to be processed, displayed, or even conceptualized. This missing number can easily be overlooked as the human consciousness will often <a href="http://www.scp-wiki.net/scp-3930">fill in the missing data</a> involuntarily. Discovery of the absent number will, however, induce some manner of mental strain and should be avoided if possible. </p>

SCP-ğ•'s relationship to <a href="http://www.scp-wiki.net/scp-033"> other anomalies</a> is under investigation.</p>

<p>The effect of exposure to SCP-ğ• appears to have some limitations, as discovered through informal and formal testing:</p>

<ul><li>When a number is given a non-numerical symbol to represent it, such as <span class="mathletter">g</span>, SCP-ğ• will still affect the underlying value, but the symbol itself will remain unaffected.</li>
<li>SCP-ğ• affects only the full number, not its individual digits. For example, if a several-digit number <i>ABCD</i> were to be affected by SCP-ğ•, only ABCD would ease to exist; the digits <i>A</i>, <i>B</i>, <i>C</i>, and <i>D</i> would not.</li></ul>
</div>

<div class="block grey" data-title="ADDENDUM: ORIGIN">
<p>SCP-ğ• was first theorized by famous mathematician â–ˆâ–ˆâ–ˆâ–ˆâ–ˆâ–ˆâ–ˆ â–ˆâ–ˆâ–ˆ in the same year the Summer Olympics were held in Atlanta, Georgia. In a mathematics essay distributed among the Mathematics department of the University of Calgary, Dr. â–ˆâ–ˆâ–ˆ theorized the following: </p>

<div class="block lblue" data-title="EXCERPT">The universe abides by rules, which are represented by the laws of physics and calculated by mathematics. There is a set of every countable thing: the number of atoms in the Empire State Building, the period between each crash of waves on the shore. If there are infinite numbers of these countable sets of things which exist, then it serves to reason there is a countable set which contains things which do not.</div>

<p>As of yet, it has not been determined whether this theory, or Dr. â–ˆâ–ˆâ–ˆ's work afterward, induced the subsequent manifestation of SCP-ğ•, but operatives in the University later uncovered the anomalous set and its effects, and developed protocols to contain it.</p>

<div class="heading"><span style="margin-left: 1em; background: #aaa; padding-left: .5em; padding-right: .5em;">NOTE</span></div>
<p>During containment, a very small amount of numbers were affected by SCP-ğ•. The total amount is unknown. <b>See section titled "SCP-ğ•-A".</b></p>
</div>
<div class="block grey" data-title="ADDENDUM: SCP-ğ•-A">
<p>RESEARCHER: DR. TREYVON BUCKHANON: During the initial containment of SCP-ğ•, a Foundation operative assigned the anomaly a standard item designation, ï¿½ï¿½ï¿½ï¿½, thinking that as long as the number was not added to the containment file, it would be safe to do so. He was wrong.</p>

<p>When this number ceased to exist, it caused extensive infrastructure outages worldwide for a short time. Foundation mathematicians and digital response teams were able to restore functionality to many systems before too long, by creating an algorithm that sort of...rounds everything upward or downward by a certain amount so computers don't notice it's missing.</p>
</div>

<div class="block purple" data-title="TESTING LOG">

<p>Testing process consisted of the following: </p>

<ul><li>Querying the RGU-ğ• system for a number consisting of an enormous amount of digits, the practical use of which is astronomically unlikely to exist at any point in the future</li>
<li>Adding or subtracting variables to this number to give it certain attributes, such as being a fraction, decimal, or have certain divisors</li>
<li>Designating this new number as a lowercase alpha character, usually <span class="mathletter">g</span></li>
<li>Allowing the RGU-ğ• system to simultaneously place it into an offsite text file as well as appending ("feeding") it to the metadata of the SCP-ğ• containment file</li>
<li>Querying the offsite file to check for data loss indicative of being affected by SCP-ğ•</li></ul>

<div class="heading"><span style="margin-left: 1em; background: #c9d; padding-left: .5em; padding-right: .5em; color: #FFF;">INITIAL LOG</span></div>
<p>Loading testing logs...</p>
<p>INPUT: <span class="mathletter">g</span> is an unmodified number. <span class="mathletter">g</span> is fed to SCP-ğ•.</p>
<p>RESULT: Full data loss detected: <span class="mathletter">g</span> ceases to exist.</p>
<p>RESEARCHER NOTE: Baseline result.</p>
<div class="heading"><span style="margin-left: 1em; background: #c9d; padding-left: .5em; padding-right: .5em; color: #FFF;">SUBSEQUENT LOG</span></div>
<p>INPUT: All the digits of one number, <span class="mathletter">gg</span>, followed immediately by the digits of a second number, <span class="mathletter">hh</span>, are concatenated into a single number, <span class="mathletter">gghh</span>. <span class="mathletter">gghh</span> is fed to SCP-ğ•. The two smaller numbers are not fed to SCP-ğ•.</p>
<p>RESULT: Partial data loss detected: <span class="mathletter">gghh</span> ceases to exist. <span class="mathletter">gg</span> and <span class="mathletter">hh</span> are unaffected.</p>
<p>RESEARCHER NOTE: It's safe to assume that constituent digits are not impacted. That's a relief.</p>
<div class="heading"><span style="margin-left: 1em; background: #c9d; padding-left: .5em; padding-right: .5em; color: #FFF;">SUBSEQUENT LOG</span></div>
<p>INPUT: <span class="mathletter">g</span> turned into an ordinal number by appending "th" to the end of it. <span class="mathletter">g<sup>o</sup></span> is fed to SCP-ğ•.</p>
<p>RESULT: Full data loss detected: <span class="mathletter">g</span> ceases to exist.</p>
<p>RESEARCHER NOTE: It appears the result is tied to the value, rather than cardinality, of the number.</p>
<div class="heading"><span style="margin-left: 1em; background: #c9d; padding-left: .5em; padding-right: .5em; color: #FFF;">SUBSEQUENT LOG</span></div>
<p>INPUT: <span class="mathletter">g</span> is described without numbers, using non-numerical words. The description, which is incredibly long, is fed to SCP-ğ•.</p>
<p>RESULT: No data loss detected.</p>
<p>RESEARCHER NOTE: It seems that if you use enough non-numbers to describe the value you're referencing, SCP-ğ• cannot affect it. This is useful for the completion of this containment file.</p>
<div class="heading"><span style="margin-left: 1em; background: #c9d; padding-left: .5em; padding-right: .5em; color: #FFF;">SUBSEQUENT LOG</span></div>
<p>INPUT: The name of an individual, whose surname is also a relatively low-value number, is to be fed to SCP-ğ•.</p>
<p>RESULT: Test not performed</p>
<p>RESEARCHER NOTE: I theorize SCP-ğ• would have been able to differentiate between when a numerical term is being used to represent a numerical value, and when it is not. But we can't take that chance.</p>
